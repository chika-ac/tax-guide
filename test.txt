# 朝礼周知システム 仕様書
Claude Code 実装用

---

## 1. システム概要

社内の朝礼で読み上げる周知文を、日付ごとに管理・閲覧するためのローカルHTML/JSONシステムです。
共有フォルダ（NAS等）に設置し、ブラウザで直接開いて使用します。サーバーや特別なソフトウェアは不要です。

### 1.1 ファイル構成

```
朝礼周知/
├── index.html              閲覧ポータル
├── editor.html             周知文の入力・保存ツール
├── calendar.html           予定マスター管理ツール
└── data/
    ├── schedule.json       予定マスターファイル
    └── YYYY-MM-DD.json     日別周知文（毎日蓄積）
```

| ファイル | 役割 |
|---|---|
| index.html | 閲覧ポータル。ブラウザで開くだけで今日の周知を表示する |
| editor.html | 周知文の入力・保存ツール。毎日の周知文を作成・追記してJSONをダウンロードする |
| calendar.html | 予定マスター管理ツール。定例・個別予定を登録してschedule.jsonを出力する |
| data/schedule.json | 予定マスターファイル。calendar.htmlで生成・管理する |
| data/YYYY-MM-DD.json | 日別周知文ファイル。editor.htmlが毎日生成し、data/フォルダに蓄積される |

### 1.2 運用環境

- 設置場所：共有フォルダ（NAS・社内ドライブ等）
- 閲覧方法：ブラウザでindex.htmlを直接開く（サーバー不要）
- 対応ブラウザ：Chrome / Edge（最新版）
- ネットワーク接続：不要（完全ローカル動作）

## 2. データ仕様

### 2.1 日別周知文 YYYY-MM-DD.json

1日1ファイル。ファイル名は日付（例：2025-04-20.json）。data/フォルダに蓄積される。

```json
{
  "date": "2025-04-20",
  "common": [
    "全員向けお知らせA",
    "全員向けお知らせB"
  ],
  "teams": {
    "営業チーム": ["営業向け連絡事項"],
    "技術チーム": ["技術向け連絡事項"],
    "総務チーム": [],
    "企画チーム": ["企画向け連絡事項"],
    "物流チーム": []
  }
}
```

備考：
- common：全員共通のお知らせ。文字列の配列。
- teams：チーム名をキーとしたオブジェクト。値は文字列の配列。
- 連絡事項がないチームは空配列[]または省略可。
- 未来日付のファイルを事前に作成しておくことができる。index.htmlは当日になったら自動表示する。

### 2.2 予定マスター schedule.json

calendar.htmlで管理する予定データ。data/フォルダに1ファイルとして保存する。

```json
{
  "recurring": [
    { "title": "全体朝礼",       "rule": "every-week",       "dow": 3 },
    { "title": "役員会",         "rule": "every-month-week", "week": 2, "dow": 1 },
    { "title": "給与締め",       "rule": "every-month-day",  "day": 25 },
    { "title": "隔週ミーティング","rule": "every-2weeks",     "dow": 5 },
    { "title": "隔月報告会",     "rule": "every-2months",    "dow": 2 }
  ],
  "events": [
    { "date": "2025-06-15", "title": "クライアントA 最終確認" },
    { "date": "2025-09-03", "title": "展示会出展" }
  ]
}
```

recurringフィールドのrule値：

| rule値 | 必須フィールド | 説明 |
|---|---|---|
| every-week | dow | 毎週。dow=曜日番号（1=月〜5=金） |
| every-2weeks | dow | 隔週。dow=曜日番号 |
| every-month-week | week, dow | 毎月第N週。week=第何週（1〜4）、dow=曜日番号 |
| every-month-day | day | 毎月N日固定。day=日付（1〜31） |
| every-2months | dow | 隔月。dow=曜日番号 |

## 3. index.html — 閲覧ポータル

### 3.1 概要

ブラウザでindex.htmlを開くと今日の周知が自動表示される。
過去分はページ下部でスクロールまたは日付ジャンプで閲覧できる。

### 3.2 動作仕様

**初期表示**
- ページを開いた日付（今日）をもとに data/YYYY-MM-DD.json をfetchで読み込む。
- 今日のJSONが存在する場合：全員共通セクションとチーム別カードを表示する。
- 今日のJSONが存在しない場合：「本日の周知はまだ作成されていません。editor.htmlから作成してください。」とメッセージを表示する。

**過去分の表示**
- 直近30日分のJSONファイルをfetchで順次確認し、存在するものを新しい順に一覧表示する。
- ページ下部に「過去の周知を確認する」セクションを設ける。
- 日付ピルボタンをクリックすると該当日にスムーズスクロールする。
- ピルと過去分コンテンツは区切り線（hr）で今日分と分離する。

**チーム別カードの表示**
- 全チームのカードを常時表示する（絞り込み機能なし）。
- 連絡事項がないチームは「本日の連絡事項はありません」と表示する。
- チームの色分け（ドット）は固定色とする。
  - 営業チーム：青 (#3a6fd8)
  - 技術チーム：緑 (#1d9e75)
  - 総務チーム：オレンジ (#d85a30)
  - 企画チーム：紫 (#7f77dd)
  - 物流チーム：アンバー (#ba7517)

### 3.3 画面構成（上から順）

1. ヘッダー：タイトル「朝礼周知」と今日の日付
2. 今日のセクション：「今日」バッジ付き日付、全員共通カード、チーム別カードグリッド
3. 区切り線：今日分と過去分を視覚的に分離
4. 過去分エリア：「過去の周知を確認する」見出し、日付ピル一覧、過去の周知文（新しい順）

### 3.4 技術仕様

- 純粋なHTML/CSS/JavaScriptで実装（フレームワーク不使用）。
- JSONの読み込みはfetch APIを使用する。ファイルが存在しない場合（404）は静かにスキップする。
- ローカルファイルからfetchを使うため、Edgeでの使用を推奨する旨をページ内またはREADMEに記載する。

## 4. editor.html — 周知文入力ツール

### 4.1 概要

管理者またはチーム担当者が毎日の周知文を入力・保存するツール。
JSONをダウンロードしてdata/フォルダに配置することで更新が完了する。

### 4.2 操作フロー（ステップ順）

**Step 1 — 日付入力**
- デフォルトで今日の日付がセットされる。
- 「今日にする」ボタンで今日の日付に戻せる。
- 未来日付を入力して事前作成も可能。

**Step 2 — 既存ファイルの読み込み（追記時のみ）**
- その日のJSONがすでにある場合、D&Dまたはクリックで読み込む。
- 読み込み後、全員共通・チーム別の内容がフォームに展開される。
- 新規作成の場合はスキップする。

**Step 3 — 予定マスターの読み込み（任意）**
- schedule.jsonをD&Dで読み込む。
- 今日の日付にヒットする予定（定例・個別）を自動抽出し、全員共通欄の先頭に展開する。
- 展開後、文言は手動で編集可能。
- 展開形式：定例は「【定例】タイトル」、個別は「タイトル」。

**Step 4 — 全員共通のお知らせ入力**
- テキストエリアで1項目ずつ入力する。
- 「＋項目を追加」で行を増やせる。
- 各行に「削除」ボタンがある。

**Step 5 — チーム別お知らせ入力**
- チーム名のタブで切り替えて入力する（タブ：営業・技術・総務・企画・物流）。
- 「＋項目を追加」「削除」は全員共通と同様。

**Step 6 — 内容確認**
- 入力内容のプレビューをリアルタイムで表示する。

**Step 7 — 保存**
- 「JSONをダウンロード」ボタンでYYYY-MM-DD.jsonをダウンロードする。
- ダウンロードしたファイルをdata/フォルダに保存（追記の場合は上書き）する。

### 4.3 チーム構成

チームは以下の5チームとする。コード上はJavaScript配列で管理し、追加・変更が容易な構造にする。

```javascript
const TEAMS = ['営業チーム', '技術チーム', '総務チーム', '企画チーム', '物流チーム'];
```

### 4.4 schedule.jsonとの連携

- 読み込んだschedule.jsonから、開いている日付にヒットする予定を抽出する。
- 抽出ロジックはセクション6の共通ユーティリティ関数を使用する。
- ヒットした予定は全員共通欄の先頭に自動入力する。

## 5. calendar.html — 予定マスター管理ツール

### 5.1 概要

定例・個別予定を登録し、schedule.jsonを生成するツール。
初期設定時と予定が増えたときに使用する。

### 5.2 個別予定の登録

**手動登録**
- 日付（date input）とタイトル（text input）を入力して「追加」ボタンで登録する。

**コピペ一括登録**
- ExcelのA列（日付）・B列（タイトル）を選択してコピーしたデータをテキストエリアに貼り付ける。
- 「一括取り込み」ボタンでパースして一覧に追加する。
- 対応フォーマット：タブ区切り（Excelコピー標準形式）。
- 日付はYYYY-MM-DD、YYYY/MM/DD、M/D、MM/DD等の一般的な形式を許容する。
- パース失敗行はスキップしてカウントを表示する（例：「8件追加、2件スキップ」）。

### 5.3 定例の登録

タイトル（text input）とパターン（select）を選択して「追加」ボタンで登録する。
パターンに応じて追加フィールドが動的に表示される。

| パターン | 追加フィールド | 表示例 |
|---|---|---|
| 毎週 | 曜日（月〜金） | 毎週水曜 |
| 隔週 | 曜日（月〜金） | 隔週金曜 |
| 毎月第N週 | 第N週（1〜4）＋曜日 | 毎月第2月曜 |
| 毎月N日固定 | 日付（1〜31） | 毎月25日 |
| 隔月 | 曜日（月〜金） | 隔月火曜 |

### 5.4 登録済み一覧

- 個別・定例を区別してバッジ表示する（「個別」「定例」）。
- 各行に「削除」ボタンがある。
- 今日にヒットする予定を緑のハイライトボックスでまとめて表示する。

### 5.5 保存

- 「schedule.jsonをダウンロード」ボタンでファイルを生成・ダウンロードする。
- ダウンロードしたファイルをdata/フォルダに上書き保存する。

## 6. 共通ユーティリティ — 予定ヒット判定ロジック

### 6.1 概要

editor.htmlとcalendar.htmlで共通して使用する、指定日付に予定がヒットするか判定する関数。
各HTMLファイルに同じコードをインラインで実装する（外部ファイル不要）。

### 6.2 関数インターフェース

```javascript
// dateStr: "YYYY-MM-DD" 形式の日付文字列
// schedule: schedule.jsonをパースしたオブジェクト
// 戻り値: ヒットした予定タイトルの配列（定例は "【定例】タイトル"、個別は "タイトル"）
function getScheduleForDate(dateStr, schedule) { ... }
```

### 6.3 判定ロジック

| rule | 判定条件 |
|---|---|
| every-week | 対象日の曜日 === dow |
| every-2weeks | 対象日の曜日 === dow かつ 基準日（2025-01-06）からの週数が偶数 |
| every-month-week | 対象日の曜日 === dow かつ その月で当該曜日がweek回目 |
| every-month-day | 対象日の日付 === day |
| every-2months | 対象日の曜日 === dow かつ 月番号が奇数（基準：2025年1月） |

備考：
- every-2weeksの基準日は2025-01-06（月曜）とする。Math.floor(weekDiff / 1) % 2 === 0 でヒット判定。
- every-month-weekの「第N週」は「その月でその曜日が何回目か」で判定する。
- every-2monthsは2025年1月を起点とし、奇数月（1, 3, 5...）をヒットとする。
- dowは1=月曜、2=火曜、3=水曜、4=木曜、5=金曜。JavaScriptのgetDay()は0=日曜のためdow+1に変換して使用する。

## 7. 更新運用フロー

### 7.1 初期セットアップ

1. 共有フォルダに「朝礼周知」フォルダを作成する。
2. index.html、editor.html、calendar.htmlを配置する。
3. data/フォルダを作成する。
4. calendar.htmlで定例・個別予定を登録し、schedule.jsonをdata/フォルダに保存する。

### 7.2 毎日の更新フロー（管理者）

1. editor.htmlを開く。
2. 日付を確認（今日の日付が自動セット）。
3. 必要に応じてschedule.jsonをD&Dで読み込み、予定を展開する。
4. 全員共通・各チームの内容を入力する。
5. 「JSONをダウンロード」でYYYY-MM-DD.jsonをダウンロードする。
6. data/フォルダにファイルを保存する。

### 7.3 チーム担当者による追記フロー

1. editor.htmlを開く。
2. 日付を入力する。
3. その日のJSONがすでにある場合、data/フォルダからファイルをD&Dで読み込む。
4. 自分のチームのタブを選んで内容を追記する。
5. 「JSONをダウンロード」でダウンロードする。
6. data/フォルダに同名ファイルを上書き保存する。

### 7.4 事前作成フロー

1. editor.htmlの日付欄に未来の日付を入力する。
2. 内容を入力してJSONをダウンロードする。
3. data/フォルダに保存しておく（例：2025-04-25.json）。
4. index.htmlは当日になると自動でそのファイルを読み込んで表示する。

## 8. ローカルファイル動作に関する注意事項

index.htmlはfetch APIでJSONファイルを読み込む。
ブラウザのセキュリティ制限により、ローカルファイル（file://）からのfetchは制限される場合がある。

**推奨対応**
- Microsoft EdgeはローカルファイルからのfetchをChomeより許容するため、Edgeでの使用を推奨する。
- Chromeの場合は --allow-file-access-from-files オプション付きで起動する。
  例：chrome.exe --allow-file-access-from-files
- または簡易ローカルサーバーを使用する。
  例：python -m http.server 8000 を実行後、http://localhost:8000/index.html を開く。
- READMEに上記の注意事項と起動方法を記載する。

## 9. 実装上の補足・制約

- 外部ライブラリは使用しない。HTML/CSS/JavaScriptのみで実装する。
- スタイルは各HTMLファイルに<style>タグで内包する（外部CSSファイル不要）。
- チーム名・チーム数はJavaScript配列で管理し、変更が1箇所で完結するようにする。
- JSONのエンコードはUTF-8とする。
- ファイルのダウンロードはBlob URLとアンカータグのdownload属性を使用する。
- index.htmlは過去30日分を対象にfetchを試みる（存在しないファイルは404で静かにスキップ）。
- index.htmlは未来日付のJSONは表示しない（今日以前のみ表示）。
- editor.htmlとcalendar.htmlは共有フォルダへの書き込みができないため、すべての保存操作はJSONダウンロード＋手動配置の方式とする。
