# 電話ログ分析ツール VBAモジュール 引き継ぎ指示書

## 概要

Excelで電話ログを分析するVBAマクロです。
「入力データ」シートにデータを貼り付け、「集計実行」マクロを実行すると
複数の集計シートが自動生成されます。

---

## ファイル構成（シート一覧）

| シート名 | 内容 |
|---|---|
| 使い方・マニュアル | 操作手順・ボタン配置 |
| 入力データ | 元データ貼り付け先 |
| イベントテーブル | 5分刻み×日付別同時通話数 |
| 受電＋折り返し分析 | 受電ごとの折り返し有無・時間 |
| 時間帯別折り返し分析 | 時間帯（1時間単位）別折り返し率 |
| 集計 | 各種集計テーブル |
| 傾向分析サマリー | KPI・ボトルネック診断 |

## 入力データ仕様

### シート名：`入力データ`
- 2行目：ヘッダー行
- 3行目以降：データ行

### 列構成

| 列 | 項目名 | 内容 | 備考 |
|---|---|---|---|
| A | 種別 | `'着信'` または `'発信'` | シングルクォート付き文字列 |
| B | 発信元電話番号 | 例：`'0454140333'` | シングルクォート付き、ハイフンなし |
| C | 受電した電話番号 | 例：`'4140333'` | シングルクォート付き、ハイフンなし |
| D | 受電日時 | 例：`'2026/04/01 08:30:55'` | シングルクォート付き、秒あり |
| E | IVR応答日時 | 同上 | 発信時は `'"'` |
| F | 通話開始日時 | 同上 | 未応答時は `'"'` |
| G | 通話終了日時 | 同上 | |
| H | 呼び出し結果 | `'応答'` `'放棄呼'` `'話中'` `'"'` | シングルクォート付き |

## 重要ルール・定義

### 電話番号照合
- **末尾7桁**で比較（シングルクォート・ハイフン除去後）
- `0454140333` と `4140333` は同一とみなす

### 除外対象行
- B列（発信元）とC列（受電先）の**両方**の末尾7桁が `4140333` の行は全集計から除外
- センター内部通話（テスト発信等）の除外が目的

### 日時パース
- シングルクォートを除去後、`yyyy/mm/dd hh:mm:ss` 形式として解析
- `'"'` は空値扱い（スキップ）

### 各種時間の定義
| 名称 | 計算式 | 単位 |
|---|---|---|
| 対応時間 | G列（終了）－F列（通話開始） | 秒→分変換で表示 |
| 待ち時間（IVR対応時間） | G列（終了）－E列（IVR応答） | 秒 |
| 折り返し時間 | 折り返し発信日時－受電日時 | 分 |

### 折り返し検索
- **同日内のみ**（翌日以降は検出しない）
- 末尾7桁で電話番号照合
- 受電より後の時刻に発信が存在すれば「折り返しあり」

### 期間絞り込み
- 全集計はイベントテーブルシートの**B3（開始日）・B4（終了日）**を参照
- 期間外のデータは全シートで除外

## VBAモジュール構成

Const NORM_DIGITS As Integer = 8      ' 折り返し照合の末尾桁数

Const TIME_END    As String  = "17:30" ' 集計終了時刻

## 集計シートの構成（Build_Aggregate）

### ① 日別対応サマリー
- 横軸：日付、縦軸：項目
- 項目：①入電件数 / ②放棄呼数 / 呼損率 / ③受電件数（①－②） / ④発信件数 / ⑤全体対応（③＋④）
- 入電件数 = 応答 ＋ 放棄呼（除外行・話中・未応答は含まない）
- 発信件数 = C列末尾7桁がセンター番号以外の発信

### ② 放棄呼テーブル（横軸=日付）
- 横軸：日付、縦軸：放棄呼数→1件目受電時刻・発信元・対応時間(秒)→2件目…
- 対応時間 = G列（終了）－E列（IVR応答）の秒数
- 列数 = `2 + nDates`（B列=項目、C列以降=各日付）

### ③ 時間別受電テーブル
- 横軸：日付、縦軸：時間帯（8時台〜17時台）
- 集計対象：応答＋放棄呼のみ（①入電件数と同定義）
- 右端に合計・構成比（%）、最下行に日付別合計

### ④ 日別総対応時間（横軸=日付）
- 横軸：日付、縦軸：着信総対応時間（時間）・発信総対応時間（時間）
- G列－F列の合計を時間単位で表示
- 列数 = `2 + nDates`

### ⑤ 日別対応時間平均（横軸=日付）
- 横軸：日付、縦軸：着信平均対応時間（分）・発信平均対応時間（分）
- G列－F列の平均を分単位で表示
- 列数 = `2 + nDates`

### ⑥ 時間帯別対応時間（平均秒）
- 縦軸：5分枠、横軸：日付
- G列－E列の平均秒数
- E列が`"`の行は除外

---

## イベントテーブルの構成

- 先頭B3・B4に開始日・終了日を入力（黄色セル）
- 8:30〜17:30の5分刻み固定（109スロット）
- A列：時間帯（HH:MM）
- B列：時間帯最高値（指定期間内の最大）
- C列以降：各日付の同時通話数（カラースケール付き）
- 右端：1時間単位の最高値サマリー

## 受電＋折り返し分析の構成

| 列 | 内容 |
|---|---|
| A | 受電日時 |
| B | 発信元番号 |
| C | 末尾7桁 |
| D | 呼び出し結果 |
| E | 通話時間（分） |
| F | 対応時間（秒）※G－E |
| G | 折り返し有無（あり/なし） |
| H | 折り返し発信日時 |
| I | 折り返し時間（分） |
| J | 短時間通話フラグ（1分未満で○） |

---

## 時間帯別折り返し分析の構成

- 1時間単位（8時台・9時台…17時台）
- 受電数・折り返し数・折り返し率・平均折り返し時間（分）・中央値・90パーセンタイル
- 指定期間全体を集計（日別ではなく期間合算）

---

## 既知の注意事項

1. **同日折り返しのみ**：翌日以降の折り返しは検出されない
2. **除外判定**：`IsSelfCall`関数でB列・C列両方がセンター番号の行を除外
3. **nColsの計算**：②④⑤は `2 + nDates`（B列=項目列のため）
4. **`GoTo`ラベルの重複禁止**：NextRow2〜NextRow8、SkipAb等は同一Sub内で一意
5. **Const配置**：モジュール先頭（Option Explicit直後）に記載、Sub/Function内部や間には置かない
6. **IIf禁止**：VBAのIIf は両辺評価のためゼロ除算が起きる。`If...Then...Else`で代替
7. **ネストSub禁止**：Sub内にSubは定義できない。全てモジュールレベルで定義

## バグ修正履歴（参考）

| 症状 | 原因 | 修正方法 |
|---|---|---|
| 「変数が定義されていません」 | `Const`をSub間に配置 | モジュール先頭に移動 |
| 「変数が定義されていません」 | 関数の戻り値代入が旧名称のまま | 関数名と戻り値代入名を一致させる |
| ゼロ除算エラー | `IIf`の両辺評価 | `If...Then...Else`に変更 |
| テーブル最終列が範囲外 | `nCols = 1 + nDates`の誤り | `nCols = 2 + nDates`に修正 |
| 集計に数字が入らない | 種別が「受電」でなく「着信」 | コード内の比較文字列を「着信」に修正 |
